C++ Primer读书笔记(一) 变量与基本类型

Posted on 2020-06-15 | In reading note |

前言

因为要做C++课程TA的原因，我被迫重新复习一遍好久没用过的C++。自从用了python，不再需要处理指针和内存，也不用考虑抽象的虚函数和多继承，虽然编程效率提高不少，但是总感觉失去了什么东西。因此，今天重新翻开《C++ primer》, 再次拥抱C++。

2.1基本内置类型

C++是严格类型的编程语言，使用一个变量前必须强调其类型。

2.1.1主要算数类型

bool 布尔型 1位 char 字符型 8位（1字节） int 整型 32位（常用） float 浮点型 6位有效数字 double 双精度浮点型 10位有效数字（常用）

除了bool变量和其他拓展变量类型，其他整型可以划分为带符号和无符号两种。

2.1.2强制类型转换

赋予一个变量不属于他类型的值，会把这个值强行转化为当前类型 例如： int x = 2.1; //实际上x的值变成了2

2.1.3特殊保留字符

换行符 \n   制表符 \t   双引号 \" 单引号 \'   反斜线 \\\    回车 \r

2.2变量

变量的声明，初始化。没什么特别

2.2.1定义

变量的声明和定义应该区分开，声明(declaration)是使得名字被程序所知，而定义(definition)则是创建与名字相对应的实体。 前面加extern并且不初始化，则为声明,没有定义。一个变量仅能被定义一次，但是可以声明无数次。

2.2.2命名规范

标识符用数字，字母以及下划线组成。 1.标识符能够体现实际含义，如index 2.变量用小写字母，用户自定义类用大写字母开头，如Class 3.多个单词组合用_进行分割 4.数字不能在第一个位置

常见保留关键字：
try(异常捕捉) union(一种类型) unsigned（非符号）
using（使用命名空间） virtual（虚态）static(静态) throw（抛异常）
goto（无条件跳转，慎用）inline（内联）namespace（命名空间）
operator（操作符）enum（枚举）

2.2.3作用域

一个变量，函数，类有他的作用域，一般用花括号包含，在作用域之外不被认知。作用域分为全局作用域(global scope)和块作用域(block scope)。内层(inner)作用域会覆盖外层(outer)作用域。

2.3复合类型

2.3.1引用(reference)

用'&'符号来定义引用类型： int ival = 1024; int &refval = ival; 引用必须被初始化！初始化的对象要相匹配，而且不能是常数。 初始化之后，refval就是ival的化身，两者绑定在一起。

2.3.2指针(pointer)

万恶之源。指针让程序员获得操作内存的能力，也让C++程序陷入bug的泥潭。用'*'*定义一个指针变量，同时也用*'*'去访问指针指向的对象。 int *ip; ip是指向int类变量的指针。定义指针时可以先不初始化。 int ip = &ival; 用'&'取址，容易与引用混淆。 指针值应该属于4种状态之一： 1.指向一个对象 2.指向紧邻对象所占空间的下一个位置 3.空指针(nullptr,NULL) 4.无效指针
